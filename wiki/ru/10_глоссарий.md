# Глоссарий — CS2 / VMP / RE

По алфавиту (латиница + кириллица вперемешку по контексту). Для тех,
кто общий реверс знает, а в CS2 или VMP — впервые.

## a2x/cs2-dumper

Rust-проект, аттачится к живому CS2 и печатает все runtime globals, все
классы + оффсеты полей, все enum'ы схемы. Upstream всего из `offsets/`
и `schema/`. Репо: [github.com/a2x/cs2-dumper](https://github.com/a2x/cs2-dumper).

## Anti-cheat

Системы Valve для детекта и блокировки читеров. **VAC** — signature-
based (client-side scan на конце матча). **Overwatch** — на review
демок. Real-time anti-cheat живёт внутри cs2.

## Anti-aim

Класс читов, крутящий исходящие angles игрока так, что модель смотрит
одним, а прицел — другим. В CS2 делается перезаписью
`CSGOInputHistoryEntryPB.view_angles` в каждом исходящем user-command.

## Build number

Внутренняя integer-версия cs2. Live-читаемо через `[engine2.dll +
dwBuildNumber]`. Также выводится из SHA-256 `client.dll` с диска.
Примеры: 14169 (2026-07-11), 14170 (2026-07-15).

## CHandle

Тип entity-handle в CS2. Packed 32-бит `{ index : 15, serial : 17 }`.
Не сырой pointer. Резолв через entity-list.

## CInButtonStatePB

Protobuf-сообщение с состоянием кнопок per user-command. Флаги: attack,
attack2, jump, duck, use, forward, back, moveleft, moveright и т.д. В
`networkbasetypes.proto`.

## Controller vs pawn

**Controller** в Source 2 — персистентный per-player объект: переживает
смерть, держит ранг / wins / статистику. **Pawn** — per-life entity:
позиция, HP, оружие. `CCSPlayerController` владеет `C_CSPlayerPawn`
через `m_hPlayerPawn`.

## CS Rating

Premier-mode Elo. 0..40000. `CCSPlayerController + 0x888` при
rank-type = 11.

## CSGOInputHistoryEntryPB

Protobuf одной записи `input_history`. ~120-байтный struct. Содержит
`view_angles` — anti-aim цель FVA.

## CSGOUserCmdPB

Внешний envelope user-command. Внутри `CBaseUserCmdPB base` + `repeated
CSGOInputHistoryEntryPB input_history`.

## CUtlSymbolLarge

Handle в intern-таблицу строк CS2. 8 байт. Не сырая строка — точка в
глобальную intern-табличку. Прямая 16-байтная запись поверх
`CUtlSymbolLarge` (типа `m_szClan`) даст мусор.

## Drift

Когда cs2 обновился и ранее валидные оффсеты больше не указывают куда
надо. Между минорными патчами — редко. Между мажорными — может быть
оптом. Детект через `verify_offsets.py`.

## dwGameRules / dwLocalPlayerController / etc.

Runtime globals CS2 — `dw*`-символы по фиксированным RVA в каждом DLL.
Указывают на синглтоны или entity-list. См. `02_offsets_подробно.md`.

## Entity list

Массив всех networked-entity. Через `[client.dll + dwEntityList]`.
Каждый слот — `CEntityInstance*`.

## FNV1a-64

Fowler-Noll-Vo hash, 64-бит, "alternate"-вариант.
`h = offset_basis; for byte b: h ^= b; h *= prime`. FVA использует
чтобы идентифицировать schema-поля по имени — см.
`06_vmp_мишени.md` с четырьмя конкретными хэшами.

## FVA / FuckVacAgain

VMP-protected anti-VAC helper плагин. Section-J — его anti-aim тело.
FNV1a-хэшированный schema-lookup для устойчивости к offset-drift'у.

## GC / GameCoordinator

Валвовский out-of-band matchmaking-сервис. С cs2 общается через свой
protobuf-канал. Обновления ранка идут туда.

## Hex-Rays

Самый ходовой x64-декомпилятор. К репо напрямую не имеет отношения,
но всплывает всегда, когда идёт разговор про реверс VMP.

## Image base

Адрес, куда OS-loader мапит DLL.
`GetModuleHandleW(L"client.dll")` — текущая image-база `client.dll`.
`dw*`-globals — оффсеты относительно неё.

## Input history

`CSGOUserCmdPB.input_history` — `repeated CSGOInputHistoryEntryPB`.
Sub-tick input state. Цель anti-aim перезаписи.

## LazyString

Deferred-string тип в CS2. `CTypeName::Get()` возвращает мусор до
materialisation LazyString'а. FVA обходит сравнением vtable-pointer'ов
напрямую.

## Manifest

Runtime schema-таблица FVA. Бинарник рядом с FVA — pre-computed
FNV1a-64 → `{ field_offset, type_hint }`. Перегружается на каждый
cs2-билд.

## MVP

Round MVP star. Персистентный счётчик — `CCSPlayerController + 0x958`.

## Pawn

См. "controller vs pawn". `C_CSPlayerPawn` для живого игрока.

## Premier

Competitive mode с числовым Elo (0..40000) вместо иконки. Rank
type = 11.

## Protobuf

Google Protocol Buffers. Wire-формат cs2 для всего. Репо зеркалит все
43 `.proto` файла из `SteamDatabase/Protobufs/csgo/`.

## QAngle

Valve-шный 3-float angle: `{ float x /* pitch */, float y /* yaw */,
float z /* roll */ }`. В protobuf'ах: `CMsgQAngle`.

## Rank type

`CCSPlayerController.m_iCompetitiveRankType` в `+0x890`. Enum: 6 =
classic competitive, 7 = wingman, 11 = premier.

## RankSpooferDriver

Kernel-driver CS2-плагин, пишущий в
`CCSPlayerController.m_iCompetitiveRanking` и т.д. — фейковый ранк.

## Ranking

Classic competitive: 0..18 (silver-1 .. global elite). Premier:
0..40000. Смысл — по rank-type.

## RVA

Relative virtual address. Оффсет от image-базы модуля. `0xB09528` =
функция CreateMove лежит в `client.dll base + 0xB09528` в билде 14170.

## Schema system

CS2-шный runtime C++-type introspection. У каждого класса + поля есть
runtime-metadata. Основа и `a2x/cs2-dumper`, и manifest'а FVA.

## SeDebugPrivilege

Windows-privilege в токене процесса, нужный для `OpenProcess` чужого
пользовательского или защищённого. Elevate shell — и оно есть.

## Section-J

Секция FVA, которая делает anti-aim angle-rewrite. Читает button-state,
wrap'ит angles в (-180, 180], пишет обратно.

## SerializePartialToArray

Virtual-метод на каждом protobuf-generated C++ классе CS2. FVA хукает
его на `CBaseUserCmdPB` для инспекции / мутации исходящих user-command.

## Signon state

Где клиент в connection-handshake. `SIGNONSTATE_FULL = 6` — "в игре".
Через `[[engine2.dll + dwNetworkGameClient] + 0x230]`.

## Source 2

Движок Valve, преемник GoldSrc / Source. Крутит cs2, Deadlock, Dota 2.

## Subtick

CS2 обрабатывает input с sub-tick-грануляцией — несколько movement-
шагов на server-tick. `CBaseUserCmdPB.subtick_moves` носит эти шаги.

## SubtickMoveStep

Одна запись в `subtick_moves`. Timestamp + button state + mouse-дельта.
Миф: FVA переписывает. Нет.

## Symbol intern table

Глобальная hash-табличка cs2 для дедупа строк. Через
`CUtlSymbolLarge`-хэндлы.

## Tick / TickBase

CS2 тикает 64 Гц. `m_nTickBase` в `CBasePlayerController + 0x6B8` —
локальная per-player tick-точка отсчёта.

## VAC

Valve Anti-Cheat. Signature-based сканер. Отложенные баны (недели) —
корреляция бана с конкретным cheat-релизом менее очевидна.

## VacLiveBypass / VLB

CS2-плагин с pattern-scan для hook-локаций (в отличие от
FNV1a-schema-подхода FVA). См. `create_move_hook.cpp` и
`serialize_to_array_hook.cpp`.

## Vtable

C++ virtual function table. FVA сравнивает vtable-pointer'ы (не
`CTypeName::Get`) для идентификации типов сообщений.

## VMProtect / VMP

Коммерческий code-virtualizer. Конвертит нативный x86 в байткод для
встроенной VM. Anti-reverse. FVA, VLB, RankSpooferDriver все
поставляются в VMP-виде.

## Wingman

2v2-competitive режим CS2. Rank type = 7.
