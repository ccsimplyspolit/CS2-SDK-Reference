# Обзор репо

## Что здесь собирается

Любому RE-проекту про CS2-бинарник — anti-VAC хелпер, aim-ассист, ранк-
спуфер, demo-парсер — нужны одни и те же 4 категории данных:

1. **Runtime globals** — `dw*`-символы, которые CS2-шные DLL экспортят
   по фиксированным RVA. Это точки входа для любой pointer-chain:
   `client.dll + dwLocalPlayerController`, `client.dll + dwGameRules`
   и т.д.
2. **Оффсеты member'ов классов** — внутри каждого экземпляра
   `CCSPlayerController`, `C_CSGameRules`, `C_BaseEntity`: с какого
   байта лежит какое поле? `m_iCompetitiveRanking`,
   `m_pInventoryServices`, `m_iHealth`, ...
3. **Wire-протобуфы** — определения всех сообщений, которые CS2
   гоняет по сети: `CBaseUserCmdPB`, `CSGOUserCmdPB`,
   `CSGOInputHistoryEntryPB` и всё остальное.
4. **Layouts структур** — читаемые summary-описания ~10 C++ классов, к
   которым RE-проект возвращается постоянно. `schema/client_dll.json`
   содержит 3189 полей в 300+ классах; для VMP-работы важна маленькая
   горстка, и репо помечает какая.

Исторически каждый проект собирал это заново. Этот репо — one-stop
версия, обновляемая из тех же upstream-источников, что использует и
всё остальное сообщество.

## Четыре upstream-источника

| Источник | Что даёт | Как забираем |
|---|---|---|
| [`a2x/cs2-dumper`](https://github.com/a2x/cs2-dumper) `output/*` | `offsets.json` + `client_dll.json` + `engine2_dll.json` + `server_dll.json` | `raw.githubusercontent.com` — `tools/sync_from_upstream.py` тянет `HEAD/main` или head-branch открытого PR. |
| [`SteamDatabase/Protobufs`](https://github.com/SteamDatabase/Protobufs) `csgo/*.proto` | 43 `.proto` файла | GitHub contents API (`api.github.com/.../contents/csgo`), потом per-file raw fetch. |
| **Живой процесс cs2** | Ground truth для сверки чего угодно | `verify_offsets.py` использует `ReadProcessMemory` + `SeDebugPrivilege`. |
| **Реверснутые VMP-сэмплы** | Маппинг "какое поле схемы соответствует какому хуку VMP-плагина" | Ручной сбор — в `vmp_targets/README.md`. |

`a2x/cs2-dumper` — Rust-проект, который аттачится к живому CS2, обходит
schema system и печатает все классы + все оффсеты полей. Когда cs2
обновляется, кто-то гонит cs2-dumper на новом билде и коммитит в
`main` или открывает PR к `a2x/cs2-dumper`. Тогда же `sync.cmd` в
этом репо начинает видеть новое.

## Чем этот репо НЕ является

- Не CS2-SDK для компиляции чего-либо. Хочешь собрать плагин к игре —
  бери Metamod:Source или Source2Mod. Этот репо — data reference.
- Не real-time сервис оффсетов. Когда cs2 обновляется и локально
  прогоняется `sync.cmd`, свежие данные лежат здесь с задержкой на
  часы (dumper turnaround). Sub-minute ground truth — только запустив
  cs2-dumper локально против своего живого cs2.
- Не аффилированно с Valve. Каждый байт здесь — из публичного реверса
  и публичных protobuf-дефинишенов.

## Связанные репозитории

- [VMP-Deob](https://github.com/ccsimplyspolit/VMP-Deob) — VMProtect
  research + тулинг. Потребляет оффсеты отсюда для runtime-верификации.
- [CS2-P2C-TEMPLATES](https://github.com/ccsimplyspolit/CS2-P2C-TEMPLATES) —
  CS2-плагины (VacLiveBypass, RankSpoofer, IsValveDS), которые
  автофетчат этот репо.
- [a2x/cs2-dumper](https://github.com/a2x/cs2-dumper) — upstream
  оффсетов + schema.
- [SteamDatabase/Protobufs](https://github.com/SteamDatabase/Protobufs) —
  upstream protobuf'ов.
