# Структуры — читаемые layouts классов

В `structures/README.md` — быстрая таблица. Здесь — контекст и пара
классов, которые верхнеуровневая страница пропускает. Все оффсеты — на
билд 14170 (2026-07-15).

## CCSPlayerController

Достаётся через `[client.dll + dwLocalPlayerController]`. Владеет
персистентным per-player state (ранг, wins, MVPs). Extends
`CBasePlayerController`.

```
offset  type                                   field
0x818   CCSPlayerController_InventoryServices* m_pInventoryServices
0x860   CUtlSymbolLarge                        m_szClan
0x888   int32                                  m_iCompetitiveRanking
0x88C   int32                                  m_iCompetitiveWins
0x890   uint8                                  m_iCompetitiveRankType
0x894   int32                                  m_iCompetitiveRankingPredicted_Win
0x898   int32                                  m_iCompetitiveRankingPredicted_Loss
0x89C   int32                                  m_iCompetitiveRankingPredicted_Tie
0x914   CHandle<C_CSPlayerPawn>                m_hPlayerPawn
0x93C   int32                                  m_iScore
0x950   int32                                  m_iMusicKitID
0x954   int32                                  m_iMusicKitMVPs
0x958   int32                                  m_iMVPs
```

### Заметки

- **`m_szClan`** — `CUtlSymbolLarge`, 8-байтный handle в intern-таблицу,
  не сырая строка. Наивные 16-байтные writes здесь дадут мусор в
  scoreboard. RankSpoofer это решает через intern-API, а не через прямую
  запись.
- **`m_iCompetitiveRanking`** — смысл зависит от rank-type
  (`m_iCompetitiveRankType` в `+0x890`):
  - Тип 6 (classic competitive): 0..18 (silver-1 .. global elite)
  - Тип 7 (wingman): 0..18
  - Тип 11 (premier): 0..40000 — CS Rating Elo
- **`m_hPlayerPawn`** — `CHandle` (packed `{ index : 15, serial : 17 }`),
  не сырой pointer. Резолви через `EntitySystem::LookupEntity(handle)`
  либо классический `(handle & 0x7FFF) → EntityList[index]`.

## CCSPlayerController_InventoryServices

Через `[CCSPlayerController + 0x818]`.

```
offset  type            field
0x58    uint32          m_unMusicID
0x5C    Rank[6]         m_rank                              // 6 service medals
0x74    int32           m_nPersonaDataPublicLevel           // Steam CS-level 1..40
0x78    int32           m_nPersonaDataPublicCommendsLeader
0x7C    int32           m_nPersonaDataPublicCommendsTeacher
0x80    int32           m_nPersonaDataPublicCommendsFriendly
0x84    int32           m_nPersonaDataXpTrailLevel
```

Каждая запись `Rank` — 12 байт `{ int32 rank, int32 month, int32 year }`.
Service medals + operation coins живут здесь.

## C_CSGameRules

Через `[client.dll + dwGameRules]`. Синглтон game-state.

```
offset  type      field
0xA0    int32     m_nQueuedMatchmakingMode
0xA4    bool      m_bIsValveDS
```

**`m_bIsValveDS` — источник gate-byte для FVA.** FVA резолвит это поле
через FNV1a-64 хеш `0xB6E8F068409FEF6C`
(= `fnv1a("C_CSGameRules->m_bIsValveDS")`), читает байт, зеркалит в свой
собственный gate-byte в `FVA_image_base + 0x23A9A0`. Секции J FVA
гейтятся по этому байту.

## C_BaseEntity

База всех entity. Все pawn'ы, оружие, world-props наследуются.

```
offset  type            field
0x34C   int32           m_iHealth               // 0 = мёртв
```

`m_iHealth` в `+0x34C` — вход fire-флага FVA, резолвится через
FNV1a-64 `0x9320AED4F2DDF04B`.

## CBasePlayerController

Родитель `CCSPlayerController`. Свои поля.

```
offset  type            field
0x6B8   int32           m_nTickBase             // live subtick counter
```

FVA читает это через FNV1a-64 `0xC08C21B68A2C746D` — выравнивает свой
тик-счётчик с cs2.

## C_CSPlayerPawn

Per-life pawn для CS2-игрока. Через `[client.dll + dwLocalPlayerPawn]`
или через `CCSPlayerController.m_hPlayerPawn`.

Полей много — всё есть в `client_dll.json`. Плагину обычно нужны:

- Inherited из `C_BaseEntity` — `m_iHealth`, `m_MoveType` и т.д.
- Velocity / позиция — CS2 держит spatial state в `CGameSceneNode`;
  `m_pGameSceneNode` пешки указывает на него.
- Оружие — `m_pWeaponServices` даёт список экипированных.

## CBaseUserCmdPB (protobuf)

Wire-envelope user-input. Экземпляры сериализуются через
`SerializePartialToArray` (virtual). Hook C FVA — на этой функции.

```proto
message CBaseUserCmdPB {
    optional int32              command_number = 1;
    optional int32              tick_count     = 2;
    optional CInButtonStatePB   buttons_pb     = 20;
    repeated CSubtickMoveStep   subtick_moves  = 18;   // FVA не трогает
    optional string             move_crc       = 27;   // FVA scratch (Hook A)
    // ...
}
```

## CSGOUserCmdPB (protobuf)

Внешний envelope — содержит `CBaseUserCmdPB` + CSGO-специфичный
input_history.

```proto
message CSGOUserCmdPB {
    optional CBaseUserCmdPB              base                          = 1;
    repeated CSGOInputHistoryEntryPB     input_history                 = 2;   // цель Section-J
    optional int32                       attack1_start_history_index   = 3;
    optional int32                       attack2_start_history_index   = 4;
}
```

Vtable RVA `0x1A1E028` (билд 14170) — FVA использует это для
type-identification, потому что `CTypeName::Get()` сломан из-за
`LazyString`.

## CSGOInputHistoryEntryPB (protobuf)

Одна subtick-запись input. Аллоцируется через
`CSGOInputHistoryEntryPB::New` (~120 B struct). FVA
disambiguating'ит factory через FNV1a-хешированный callsite: сигнатура
`B9 78` (inlined 120-байтовый struct-size mov) матчит несколько
кандидатов.

```proto
message CSGOInputHistoryEntryPB {
    optional CMsgQAngle    view_angles         = 2;   // ← FVA пишет сюда
    optional CMsgVector    cl_interp           = 3;
    optional int32         render_tick_count   = 4;
    // ...
}
```

## CInButtonStatePB (protobuf)

Состояние кнопок per user command.

```proto
message CInButtonStatePB {
    optional int64  changed = 1;
    optional int64  held    = 2;
    optional int64  pressed = 3;
}
```

FVA читает бит `attack` — режим fire-gate. VLB сверху накидывает
`attack_pressed || button_state_changed` — политика построже.

## Паттерны доступа

Каждый плагин пишет одни и те же три хелпера. Copy-paste скетчи:

```cpp
// 0. База client.dll
uintptr_t client_base = (uintptr_t)GetModuleHandleW(L"client.dll");

// 1. Локальный player controller
uintptr_t lpc_ptr = client_base + 0x237EBA0;      // dwLocalPlayerController
uintptr_t lpc = *(uintptr_t*)lpc_ptr;
if (!lpc) return; // ещё не в игре

// 2. Персистентная статистика
int rank = *(int*)(lpc + 0x888);
int wins = *(int*)(lpc + 0x88C);

// 3. Inventory services + Steam level
uintptr_t inv = *(uintptr_t*)(lpc + 0x818);
int cs_level = 0;
if (inv) cs_level = *(int*)(inv + 0x74);

// 4. Game rules + gate-byte
uintptr_t gr_ptr = client_base + 0x23A39D8;       // dwGameRules
uintptr_t gr = *(uintptr_t*)gr_ptr;
uint8_t is_valve_ds = gr ? *(uint8_t*)(gr + 0xA4) : 0;
```

## Дисциплина обновления страницы

Когда cs2 апдейтится:

1. Сверь каждый hex на этой странице с `schema/client_dll.json`.
2. Правь ТОЛЬКО те числа, что реально сдвинулись.
3. Если TRACKED поле двинулось — добавь строку в
   `vmp_targets/README.md` "Recent verifications" с датой + PID.
4. Обнови строку "current as of build …" в header.
