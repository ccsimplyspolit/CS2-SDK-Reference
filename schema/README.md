# CS2 schema — class field offsets

---

## English

Machine-readable dumps of every class' member offsets inside CS2's C++ code. Auto-generated by [a2x/cs2-dumper](https://github.com/a2x/cs2-dumper) from a running CS2 process via schema-system introspection.

### Files

One JSON per module cs2-dumper emits — **18 modules, 3330 classes, 16595 fields**. The three you'll touch most:

| File | Size | Contents |
|---|---|---|
| `client_dll.json` | ~214 KB | 563 classes, 3189 fields (`CCSPlayerController`, `C_CSGameRules`, `C_BaseEntity`, etc.) |
| `server_dll.json` | ~495 KB | 990 classes, 5871 fields (large — CS2 server has more logic; holds its own `CCSPlayerController` at different offsets than the client one) |
| `engine2_dll.json` | ~11 KB | Engine-side classes |

Plus `animationsystem_dll`, `particles_dll`, `soundsystem_dll`, `pulse_system_dll`, `vphysics2_dll`, `worldrenderer_dll`, `materialsystem2_dll`, `schemasystem_dll`, `rendersystemdx11_dll`, `resourcesystem_dll`, `scenesystem_dll`, `networksystem_dll`, `host_dll`, `panorama_dll`, `steamaudio_dll`. `tools/sync_from_upstream.py` discovers whatever `*_dll.json` upstream ships, so new modules land here automatically.

### Reading

Structure (every file):

```json
{
  "client.dll": {
    "classes": {
      "CCSPlayerController": {
        "parent": { "name": "CBasePlayerController", ... },
        "fields": {
          "m_iCompetitiveRanking": 2176,
          "m_iCompetitiveWins": 2180,
          ...
        }
      },
      ...
    },
    "enums": { ... }
  }
}
```

**Query example (Python):**

```python
import json
c = json.load(open('client_dll.json'))
fields = c['client.dll']['classes']['CCSPlayerController']['fields']
print(hex(fields['m_iCompetitiveRanking']))  # 0x888
```

### Drift discipline

Between minor CS2 patches, **class field offsets rarely change**. Between major patches (schema restructures), they can move wholesale. Verify against `../offsets/history/` before assuming an old offset still holds.

**Verified: build 14169 → 14170 (2026-07-11 → 2026-07-15)** — **0 field offset changes** across all 3189 client.dll fields. Only 1 anonymous inner struct name hash changed. Cross-checked 2026-07-19: 10381 field offsets across 15 modules match the installed game exactly (`tools/verify_offsets_static.py` for globals, source2gen dump for fields).

### Regenerate

```powershell
# Preferred — clone cs2-dumper, run it on your live cs2
git clone https://github.com/a2x/cs2-dumper.git
cd cs2-dumper
cargo run --release
copy .\output\client_dll.json ..\CS2-SDK-Reference\schema\client_dll.json
```

Requires: Rust toolchain, a running CS2 with elevated shell (SeDebugPrivilege).

## Русский

Machine-readable дампы member-оффсетов каждого класса в C++-коде CS2. Автоматически генерируются [a2x/cs2-dumper](https://github.com/a2x/cs2-dumper) из running-процесса CS2 через schema-system introspection.

### Файлы

По одному JSON на каждый модуль, что выдаёт cs2-dumper — **18 модулей, 3330 классов, 16595 полей**. Три, к которым обращаешься чаще всего:

| Файл | Размер | Содержимое |
|---|---|---|
| `client_dll.json` | ~214 KB | 563 класса, 3189 полей (`CCSPlayerController`, `C_CSGameRules`, `C_BaseEntity` и др.) |
| `server_dll.json` | ~495 KB | 990 классов, 5871 поле (крупный — на сервере логики больше; держит собственный `CCSPlayerController` по другим оффсетам, чем клиентский) |
| `engine2_dll.json` | ~11 KB | Engine-side классы |

Плюс `animationsystem_dll`, `particles_dll`, `soundsystem_dll`, `pulse_system_dll`, `vphysics2_dll`, `worldrenderer_dll`, `materialsystem2_dll`, `schemasystem_dll`, `rendersystemdx11_dll`, `resourcesystem_dll`, `scenesystem_dll`, `networksystem_dll`, `host_dll`, `panorama_dll`, `steamaudio_dll`. `tools/sync_from_upstream.py` подхватывает любой `*_dll.json`, что публикует upstream, так что новые модули появляются здесь автоматически.

### Как читать

Структура (у каждого файла):

```json
{
  "client.dll": {
    "classes": {
      "CCSPlayerController": {
        "parent": { "name": "CBasePlayerController", ... },
        "fields": {
          "m_iCompetitiveRanking": 2176,
          "m_iCompetitiveWins": 2180,
          ...
        }
      },
      ...
    },
    "enums": { ... }
  }

```

**Пример запроса (Python):**

```python
import json
c = json.load(open('client_dll.json'))
fields = c['client.dll']['classes']['CCSPlayerController']['fields']
print(hex(fields['m_iCompetitiveRanking']))  # 0x888
```

### Дисциплина drift

Между минорными патчами CS2 **оффсеты member'ов классов почти не двигаются**. Между мажорными реструктурами схемы могут смениться массово. Прежде чем полагаться на старый оффсет — проверь `../offsets/history/`.

**Verified: билд 14169 → 14170 (2026-07-11 → 2026-07-15)** — **0 сдвигов field offset'ов** по всем 3189 полям client.dll. Изменился только 1 хэш имени анонимной внутренней структуры. Пересверено 2026-07-19: 10381 field offset по 15 модулям совпадают с установленной игрой точно (`tools/verify_offsets_static.py` для globals, дамп source2gen для полей).
