# CS2 schema — class field offsets

---

## English

Machine-readable dumps of every class' member offsets inside CS2's C++ code. Auto-generated by [a2x/cs2-dumper](https://github.com/a2x/cs2-dumper) from a running CS2 process via schema-system introspection.

### Files

| File | Size | Contents |
|---|---|---|
| `client.dll.json` | ~214 KB | ~200 classes, ~3189 fields (`CCSPlayerController`, `C_CSGameRules`, `C_BaseEntity`, etc.) |
| `engine2.dll.json` | ~11 KB | Engine-side classes |
| `server.dll.json` | ~495 KB | Server-side classes (large — CS2 server has more logic) |

### Reading

Structure (each of the 3 files):

```json
{
  "client.dll": {
    "classes": {
      "CCSPlayerController": {
        "parent": { "name": "CBasePlayerController", ... },
        "fields": {
          "m_iCompetitiveRanking": 2176,
          "m_iCompetitiveWins": 2180,
          ...
        }
      },
      ...
    },
    "enums": { ... }
  }
}
```

**Query example (Python):**

```python
import json
c = json.load(open('client_dll.json'))
fields = c['client.dll']['classes']['CCSPlayerController']['fields']
print(hex(fields['m_iCompetitiveRanking']))  # 0x888
```

### What every RE project needs from here

For an "unhackable" cheat-sheet of the ~10 fields any CS2 project cares about, see [`../structures/README.md`](../structures/README.md).

For the specific fields VMP-protected plugins touch, see [`../vmp_targets/README.md`](../vmp_targets/README.md).

### Drift discipline

Between minor CS2 patches, **class field offsets rarely change**. Between major patches (schema restructures), they can move wholesale. Verify against `../offsets/history/` before assuming an old offset still holds.

**Verified: build 14169 → 14170 (2026-07-11 → 2026-07-15)** — **0 field offset changes** across all 3189 fields. Only 1 anonymous inner struct name hash changed.

### Regenerate

```powershell
# Preferred — clone cs2-dumper, run it on your live cs2
git clone https://github.com/a2x/cs2-dumper.git
cd cs2-dumper
cargo run --release
copy .\output\client_dll.json ..\CS2-SDK-Reference\schema\client_dll.json
```

Requires: Rust toolchain, a running CS2 with elevated shell (SeDebugPrivilege).

## Русский

Machine-readable дампы member-оффсетов каждого класса в C++-коде CS2. Автоматически генерируются [a2x/cs2-dumper](https://github.com/a2x/cs2-dumper) из running-процесса CS2 через schema-system introspection.

### Файлы

| Файл | Размер | Содержимое |
|---|---|---|
| `client.dll.json` | ~214 KB | ~200 классов, ~3189 полей (`CCSPlayerController`, `C_CSGameRules`, `C_BaseEntity` и др.) |
| `engine2.dll.json` | ~11 KB | Engine-side классы |
| `server.dll.json` | ~495 KB | Server-side классы (крупный — на сервере логики больше) |

### Как читать

Структура (у каждого из 3 файлов):

```json
{
  "client.dll": {
    "classes": {
      "CCSPlayerController": {
        "parent": { "name": "CBasePlayerController", ... },
        "fields": {
          "m_iCompetitiveRanking": 2176,
          "m_iCompetitiveWins": 2180,
          ...
        }
      },
      ...
    },
    "enums": { ... }
  }
}
```

**Пример запроса (Python):**

```python
import json
c = json.load(open('client_dll.json'))
fields = c['client.dll']['classes']['CCSPlayerController']['fields']
print(hex(fields['m_iCompetitiveRanking']))  # 0x888
```

### Дисциплина drift

Между минорными патчами CS2 **оффсеты member'ов классов почти не двигаются**. Между мажорными реструктурами схемы могут смениться массово. Прежде чем полагаться на старый оффсет — проверь `../offsets/history/`.

**Verified: билд 14169 → 14170 (2026-07-11 → 2026-07-15)** — **0 сдвигов field offset'ов** по всем 3189 полям. Изменился только 1 хэш имени анонимной внутренней структуры.
